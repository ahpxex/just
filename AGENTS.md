# just — Agents Guide

本文档是 `just` 这款写作软件的**概念层纲领**。任何后续的设计、实现、重构、扩展都必须以本文件为最高优先级约束。当实现细节与本文件冲突时，以本文件为准。

---

## 一、产品定位

`just` 是一款**纯粹的写作软件**。

- 它只做两件事：**写作** 与 **写作管理**。
- 除此之外的一切功能都**不被允许存在**——不做笔记工具、不做知识库、不做大纲工具、不做双链、不做待办、不做协作、不做 AI 助手、不做导出管道、不做插件系统。
- "极简" 不是风格上的点缀，而是**产品的第一性原则**。每增加一项非写作功能，都是对产品的背叛。

## 二、美学基调

软件的视觉与交互风格必须呈现**东方美学的禅意**：

- **留白优先**：界面的主体是空白，文字是空白中浮现的墨迹。
- **去装饰化**：不使用渐变、阴影、拟物、磨砂、荧光、动效炫技。
- **单色克制**：以墨、宣纸、远山为基调。颜色数量极少，对比柔和。
- **呼吸感**：行高、字距、边距要宽敞，让文字有喘息的空间。
- **静默**：没有弹窗、提示音、徽章、红点、角标。
- **时间感**：若有动效，必须缓慢、自然，如同毛笔落纸的晕染，绝不跳跃、弹性、惊扰。
- **字体选择**：正文优先使用衬线或书法气质的中文字体，英文配以典雅衬线体。避免几何无衬线字体的科技感。

参考意象：宣纸、水墨、枯山水、砚台、一盏油灯、庭院深深、雪落无声。

## 三、功能范围（白名单）

只有以下功能被允许存在：

1. **写作**
   - Markdown 原生支持（标题、强调、引用、列表、代码块、链接、图片）。
   - 所见即所写：编辑与渲染合为一体，不做预览/编辑分栏。
   - 本地优先，所有文稿以纯文本 `.md` 文件形式存储在用户机器上。

2. **写作管理**
   - 文稿的新建、打开、重命名、删除。
   - 文稿列表的浏览与搜索。
   - 文稿的基本元数据（创建时间、修改时间、字数）。

**除此之外不允许存在的能力（示例，非穷举）**：设置面板中的主题切换、字体切换、云同步、账号系统、导出为 PDF/DOCX、插入表格/公式、图床、版本历史、专注统计、打字音效、番茄钟、AI 续写、语法检查、翻译、分享链接。

## 四、行为约束（强制）

这是 `just` 与普通写作软件最根本的区别。软件启动后，用户进入一种**不可逃离的写作状态**：

1. **自动全屏**：软件启动后立即进入全屏模式，无过渡页、无欢迎页、无引导。
2. **窗口不可关闭**：顶部的关闭/最小化/最大化按钮全部移除或禁用。
3. **窗口不可退出**：
   - 禁用 `Cmd+Q` / `Alt+F4` / `Cmd+W` 等常规退出快捷键在应用内的效果。
   - 禁用 Dock/任务栏的右键退出。
   - 禁用菜单栏的 Quit 项（或直接不提供菜单栏）。
4. **窗口不可切换**：尽最大能力阻止用户通过常规快捷键切换到其他窗口（在操作系统允许的范围内）。注意：不得违反 macOS/Windows 的辅助功能与系统级强制退出机制（如强制退出对话框、电源键、`Cmd+Option+Esc`），这是操作系统的最后保障，不得也无法绕过。
5. **全局启动器屏蔽**（macOS + Windows）：**策略是 deny-by-default** —— 任何带修饰键（macOS: Cmd/Ctrl/Opt；Windows: Ctrl/Alt/Win）的按键事件默认被吞掉，**只放行写作必需的白名单**。这保证无论用户用什么启动器（Raycast / Spotlight / Alfred / 自定义 hotkey），只要落在修饰键组合上就被拦住。
   - 白名单（通用思路）：
     - 文本编辑：`Cmd/Ctrl + {A, C, V, X, Z, Y}` 和 `Cmd/Ctrl + Shift + Z`
     - app 命令：`Cmd/Ctrl + {F, N, K}`
     - 导航/删除：`Cmd/Ctrl + {arrows, Backspace, Delete, Home, End, PageUp/Down}`
     - Option + {arrows, Backspace, Delete, 字母/数字/标点}（按词导航 + 重音符号输入）
     - Shift 单修饰 + 任何键（选择、大写）
     - `Cmd+Option+Esc` 强退对话框（OS 最后保障，必须保留）
     - `Cmd+Option+Q`（仅 DEV 构建，开发退出）
   - macOS：CGEventTap @ `kCGSessionEventTap`。**需要辅助功能权限**——首启触发系统权限对话框，用户"系统设置 → 隐私与安全 → 辅助功能"勾选 `just` 后，重启 app 生效。未授权时降级（tap 安装失败，app 正常跑但启动器能突破）。
   - Windows：`SetWindowsHookExW(WH_KEYBOARD_LL)` + `GetAsyncKeyState` 查询实时修饰键状态。无需额外权限。
   - Linux：不实现。
   - **Focus-aware**：`Tauri::WindowEvent::Focused` 事件驱动一个 `AtomicBool`，只在 `just` 处于前台焦点时真正拦截。用户切到系统设置授权、处理系统通知等场景下，其他 app 里的 Cmd+C 等操作不受影响。
6. **42 分钟 session 仪式后的合法出口**：完成一个 42 分钟的连续写作 session 后，出现仪式对话框，三个合法动作之一是 `leave`——此时允许真正退出 app。对话框关闭后若用户未按任何写作键，右上角保留 `leave` 按钮作为 grace 出口；用户一旦继续写（写作键触发），grace 关闭，锁定到下一个 session 结束。
7. **唯一的出口是写作本身**：用户打开 `just` 之后，唯一能做的就是不断写下去。

### 关于"不可退出"的工程伦理

"不可退出"是一个强设计主张，它服务于产品哲学，而不是对用户的劫持：

- 用户安装 `just` 即意味着**主动签署了这份契约**。
- 必须在首次启动时（或安装说明中）**明确告知**用户这一行为，不做欺骗。
- 系统级的最后退路（强制退出、关机、拔电源）不得干预，也无法干预。
- 不得做任何形式的数据绑架：所有文稿必须是本地明文 `.md`，用户随时可以用其他工具访问自己的文字。

## 五、技术基线

- **框架**：Tauri 2（已搭建）+ React 19 + TypeScript + Vite。
- **包管理**：`bun`（不要使用 `npm`）。
- **状态管理**：Jotai（原子化，避免 context 嵌套）。
- **样式**：Tailwind CSS v4（通过 `@tailwindcss/vite`，CSS 优先的配置方式）+ `@tailwindcss/typography`。不使用任何 UI 组件库。
- **编辑器核心**：CodeMirror 6 + `@codemirror/lang-markdown`。以内联渲染（inline rendering）的方式达成"所见即所写"，不做分栏预览。
- **字体**：霞鹜文楷 LXGW WenKai（正文中文），EB Garamond 或系统衬线（正文英文）。字体文件本地打包，不走 CDN。
- **Tauri 插件**：`plugin-fs`（文件读写）、`plugin-dialog`（首启选目录）、`plugin-os`（平台判断）。
- **平台特化**：
  - macOS：`objc2-app-kit` 设置 NSApp presentation options（隐 Dock/菜单栏、禁 Cmd+Tab）；`core-graphics` 装 CGEventTap 拦系统启动器；`ApplicationServices` 的 `AXIsProcessTrustedWithOptions` 触发辅助功能权限请求。
  - Windows：`windows` crate 装 `WH_KEYBOARD_LL` 低级键盘钩子拦 Win / Alt+Tab / Alt+Space。
- **Rust 侧**：负责窗口生命周期、全屏模拟、退出拦截、文件系统读写、统计持久化、全局热键屏蔽。
- **前端**：只负责渲染与交互，不承担任何窗口控制逻辑。
- **存储**：本地文件系统，文稿目录由用户在首次启动时选定（或默认到系统文档目录下的 `just/`）。
- **无网络依赖**：软件不应发起任何出站网络请求。

## 六、交互原则

- **少即是多**：任何一项新交互，先问"不做它会死吗？"，答案不是"会"就不要做。
- **快捷键克制**：只保留最本质的——`Cmd+N` 新建、`Cmd+O` 打开文稿列表、`Cmd+S` 保存（或自动保存后去掉它）、`Cmd+F` 文内搜索。
- **无模态对话框**：确认、提示、错误，都以极轻的方式呈现，最好以文本本身的状态变化来表达。
- **无设置**：没有设置面板。所有决策由设计者做完，用户只需要写。如果真的需要极少数可配置项（如文稿目录），放在一个隐藏的文本配置文件里，不做 UI。

## 七、命名与文案

- 产品名保持小写：`just`。
- 所有界面文案极简，接近无。能用一个符号表达就不用一个词，能用留白表达就不用符号。
- 如需文字，优先中文，语气克制，避免感叹号、口语化、营销化表达。

## 八、开发纪律（给 Agent 的硬约束）

当你作为 Agent 为本项目编写代码时：

1. **不要引入任何违反上述白名单的功能**，即使"顺手就能加"。
2. **不要为了演示而堆砌动效、配色、组件库**。本项目禁止使用任何 UI 组件库（Ant Design、MUI、shadcn 等），一切视觉自建。
3. **不要添加可选项/设置项**来"让用户决定"。设计者必须做完所有决策。
4. **不要生成注释解释显而易见的代码**，只在有非显而易见约束时写一行注释。
5. **不要创建 README 之外的文档文件**，除非用户明确要求。
6. **遇到与本文件冲突的请求时，先停下来和用户澄清**，不要擅自取舍。
